# MAILTOPIA

连接天体、互联万物，不能想象如今的地球互联网能在太空时代发挥作用。人类对天体的探索、开发需要一个与之相适应的信息系统，如果有人愿意参与其中，一起把这个工程以开源协作方式编制完成，可以获得免费取得[产品](https://github.com/)代理权的政策。

作为长期工程，初始协议和远期规划需要反复论证，谨以不成熟的白皮书供探讨。

指南：白皮书1（前端与功用）    白皮书2（共识及后端基础）

需求：
前端：在一个称为“心流”的智能层中运行，每次交互的实行界面由AI调取，瞬时渲染出来
后端：基于要约制定与合约履行，以电子邮件通讯为存证锁定为底层共识系统
系统：运行在一个以地球（母星）为地线端，可无限延展的、实现数据“接地”的共识信息网

要求：
前端：2002年我编写界面时，并没有人工智能这类应用，当时我用ASP编写了初版，可由主人自行决定内容开放与闭合的邮件系统，但受限于电子邮件协议的桥接能力，最后开发的成果功能十分有限，我就把它作为网站心情留言板了。2015年，有感于互联网的乱七八糟，我重新开始这个项目，在多个树莓派2B搭建的简易集群（用树莓派代表每个星球或公司）中测试通过，但是由于目标过于宏大、想法过多，兼之程序粗糙，遂中途放弃。2022年自家公司获得高新企业资质之后，我还把它作为公司成果宣传过一阵，反应寥寥，自己也觉得没意思，遂再次放弃。如今把它再度拾起，只能说是初心不改。

我还是按照场景来说明吧：

邮件发往电灯，要求”开灯“，电灯打开，并回复主人”灯已打开“；

打印机分配邮址接入系统，无需人为安装驱动（让人安装驱动的行为，真的很烦，也反人类），设备握手时自动提交驱动包以邮件形式传递给系统，系统自动读取邮件，把它匹配入系统，实现真正的即插即用；

发文档给打印机，打印机自动从内容中读取文件，执行打印。完毕，通知主人”已打印完成，输出无误“；

一个主人关联多个设备邮箱，像在微信群一样和设备闲聊暨发布任务，还可以写邮件为设备分配任务以及替设备编程。每个设备都直连厂家电子邮箱，有问题设备自行和厂家沟通解决，用户不用多操心。

厂商定期发送升级包邮件给设备，实现自动升级。设备与设备之间也会自行沟通，配合一项需要相互协作的任务，完成好后群内通知主人完成。

人与物互联，和物聊天；物与物互联，相互发送指令，囊括要约/合约/相互做买卖取得收益；物与人，人与人互通同上述。

机器与人，人与人，人与物之间所有的待办、沟通与承诺，都会作为要约发起、合约履行、执行后反馈，以共识方式传递全网（有点像区块链），不存在失诺、扯皮、违规现象。

2015年并没有现在的AI大模型，只有神经卷积之类的东西。我当时想，把所有人的电子邮件连接这个系统，用人们的邮件库作为日常语料来形成算法，让智能邮箱能读懂语音语义，这样就能写邮件给机器，让机器从简单的口语指示做起，慢慢越来越聪明。后来chatgpt 3.0出来，我一下就觉得，“咦，这不就是我当年想要的吗！”，今年OpenClaw出来，我又“咦，这不是有点接近我当时想要的结果吗？”只是，现在的Claw还没有人把它彻底引导往万物互联方向靠拢，只是提升了电脑桌面的在互联网生产力。因此我想，现在就提出规划，让大家能把Claw引导上物联网，引导上太空开发，这样其能力或大大扩增。（其实，按照现在技术，MAILTOPIA里面其实自带Claw，它只是在一个互联互通全无障碍的信息系统内发挥更大作用。只要设想，万物互联的信息量比人类数字社交社会产生出的可远远大了几个数量级不止，并且日志更规整、明确。）

根据人生的实践，我其实早已放弃在地球上实现万物互联的想法，因为那是根本是不可能的事情，因为商业与人性都是要求割据与自家独大。因此，我想天体是个机会，以月球为例：它空白，参数极简，从头开始规划好万物互联，从探索开发中把数据积垒起来，反过来就可以把地球数据重新规整，说难听些是倒逼。而太空中有无数的星辰，地球信息远低于一个像素，身处其中的人们认为海量的地球数据，其实被一口吞下，像沙子一样简易。

但是skill这个东西给我的启发不小，早年确实没想到，应该补上去。它很重要，它们会像基因片段一样把人类的各种知识经验技能如同遗传全部整合到共识链中去，这些skills会应付天体探索和建设中遇上的各种问题，并且还会不断自我进化升级，让Claw的能力不断倍增。

强调：本系统开发时务必要确定人类的共识永远高于机器，
