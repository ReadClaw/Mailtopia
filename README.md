# @MAILTOPIA

请原谅，这是一个文科生提交的代码项目。就像一些理工同学偶尔也写诗歌一般，无论好与不好，文喵听了不出声表示，理汪宽容则个。

连结星辰、互联万物，不可想象在地球上运行的互联网放入太空仍具有效用，人类理应对天体探索要求一个与之适应的新信息系统。若开发人员有意参与其中，一起把这项工程以开源协作的方式进行，可以免费得到特殊的宇航事业权属与新产品(https://github.com/)的代理权。

作为长期工程，初始协议和远期规划需要反复论证，谨以此不成熟的白皮书先行探讨，以臻完善。

指南：[白皮书1（涉及前端与功能）](https://github.com/ReadClaw/MAILTOPIA/blob/main/whitepaper/whitepaper1(AGPL-3.0%20license).zip) | [白皮书2（关于共识及底层）](https://github.com/ReadClaw/MAILTOPIA/blob/main/whitepaper/whitepaper2.doc)

需求：
前端：在一个称为“心流”的智能层中运行，每次交互的实行界面由AI调取，瞬时渲染出来
底层：基于要约制定与合约履行，以电子邮件通讯为存证锁定为底层共识系统
系统：运行在一个以地球（母星）为地线端，可无限延展的、实现数据“接地”的共识信息网


从说明过往了解项目：2002年我尝试编写，当时并没有人工智能这类应用，就用ASP编写了初版“一种可由使用者自行决定内容范畴开放与闭合的邮件系统”。可限制于电子邮件协议的知识能力，最后开发成的功能十分局限，我就把它转为一种基于电邮的留言板了。2015年，有感于互联网的乱七八糟，我重新拾掇这个项目，在多个树莓派2B搭建的简易集群中测试通过，并命名为MAILTOPIA。我用树莓派做多个邮局网关示意为各个星球和宇航公司，设想铺建一种跨星球的通讯，但由于目标过于宏大、想法过多，兼之程序粗糙，遂中途放弃。2022年自家公司获得高新企业资质之后，我还把它作为公司成果宣传过一阵，反应者寥寥，自己也觉得没意思，遂又放弃。如今把它再度拣起，只能说是初心不改。

这事情，其实和OpenClaw还挺有关系，但并不是"非得计算机专业的才可以编写代码" 这种跨度让我看到曙光，毕竟对于一个要面向宇宙的操作系统来说，全部代码交由大模型Vibe Coding生成，我想是绝对不正确的。而是，从前在MAILTOPIA的运行模式中包含有类似Claw这样的智能化表达，却又不是Agent。而且MAILTOPIA欲达成目标是真实万物互联的，因此它理应包含能直接操控和对话物理世界的各种设备，因而能把力量从桌面系统延伸到了现实。


我还是按照场景举例说明，会更易于理解：

万事万物分配邮址，跨域通讯，在应用层中，人与物平权。

邮件发往电灯，要求“开灯”，电灯打开，并回复主人“灯已打开”。

打印机分配邮址接入系统，无需人为安装驱动（让人安装驱动的行为，真的很烦，也反人类），设备握手时自动提交驱动包以邮件形式传递给系统，系统自动读取邮件，把它匹配入系统，实现真正的即插即用。

发文档给打印机，打印机自动从内容中读取文件，执行打印。完毕，通知主人“已打印完成，输出无误”。

一个主人关联多个设备的邮箱，像在微信群一样和设备闲聊暨发布任务，还可以写邮件为设备分配任务以及替设备编程。每个设备都直连厂家电子邮箱，有问题的设备自行和厂家沟通解决，用户不用过多操心。

给一个合作公司发邮件，其中包含要回传的合同，系统自动检测，并协助抽取出不同合同条款，分往各不同的涉及部门审阅，而后再拼合回已修正的完整合同，盖戳回传；采购单也同理，拆解分往不同专员（不一定是人）、各生产商邮件处理。

厂家定期发送升级包邮件给设备，实现自动升级。设备与设备之间也会自行沟通，领取需要相互协作的各自任务，完成好后邮件群内通知主人完成。

汽车的每个零部件都可以分配邮址，追溯至生产端事件邮址，总线实为车体的内部沟通群，汇总生成实时仪表盘给用户。

传感器所处的环境和电气指标，生成日志邮件给设备管理员和厂家，让有权限者共同监测，随时调用。非权限人员则可通过邮址浏览器调阅公共参数，发布建议。以此协作，没有APP阻隔，万物在生长。

人与星球对话，点亮整个星球。星球与星球对话，火星要求接入地球申请作为临时硬盘。八大行星联合发布一次社交动态，即将排成一字横模式，哈勃天文镜自动回复，“相机就位，马上安排”。

界面是瞬时渲染生成，万事万物都可以即时渲染成前端界面与后台界面，切分邮件内容为“对外可观看部分”与“后台权限可查看部分”（对于人类，则类似于把社交分为“公开”与“隐私”）。

人与物互联，和物聊天；物与物互联，相互发送指令，囊括要约/合约/相互做买卖取得收益/结算分配；物与人，人与人互通同上述。

机器与人，人与人，人与物之间所有的待办、沟通与承诺，都会作为要约发起、合约履行、执行后反馈，以共识方式传递天体全网（有点像区块链），流程附戳，不存在失诺、扯皮、违约现象。

无法尽述场景，皆因数据无壁垒，请随意想象，我把mailto协议与topia两个词汇组合成mailtopia。设想一个受人类共识制约的物联网，它会自动遵循多数人和平发展的意愿建设地球；它会在流星来袭时依从人类恐慌的自然天性，发射出目标统一的导弹群并紧急采纳各项措施阻止灾难、保护人类；它会遂人们整体意愿，自发调动整个太阳系的所有设备，如同一个积极的强壮巨人一样，向天体开发的大战略持续奋战。因此，所有通道必须给系统一个“地线”，以信息“接地”方式约束人工智能，把机器智慧和硬件掌控在主人手中。这样可以阻止部分见异思迁的人类、压制反群体共识的思想，以星际共识狙击异变，遏制战争，并实现老生常谈的公平性。

如此流畅敏捷，没有app与复杂层级，越简单的网络结构逻辑越通顺，实现全人类共同体的协作。我也几乎肯定确信，这样的系统在未来能通过人工智能一键编制完成，但恐代码成为黑盒，因此强烈建议人们依靠自身完成，AI则为辅，以开源实现代码审查。2015年囿于未有现如今的AI大模型，只有神经卷积之类的技能，也远无达到“涌现”地步。我当时的想法，把所有人的和机器的电子邮件连接这个系统，用这个邮件库作为日常语料来形成算法匹配，让邮箱智能化后能初步读懂语音语义，这样就能写邮件给机器，让机器从简单的口语指示做起，初始达到像小冰、siri这样的水平，再争取越来越聪明和泛化。再后来chatgpt 3.0出现，确让人一下诧异，“咦，投喂还能出这种效果！”。今年OpenClaw出来，我又，“咦，这不就是要求的那种智能插件效果吗？”彼时我的预想是MAILTOPIA有了智能化以后，有一种插件在渲染出的界面自动操作给人看，便于主人监督对机器的控制。只是，现在的Claw还没有人把它彻底引导往万物互联方向靠拢（因地球上根本没有万物互联），仅提升了电脑桌面的在互联网生产力。因此我想，应当做出规划，让大家能把Claw引导上物联网，引导上太空开发，这样其能力与用途就大大扩增了。另外，不可想象的，现在的Claw的多数步骤是先以截图然后再识别方式，并依赖一堆API去开展工作，而非穿行于比特间。最后我要说，真正能实现万物互联走出地球的只有MAILTOPIA，因为：

1.它是真实实现跨域的系统。共识基础A：邮件是互联网的第一个应用，人们对email的使用习惯和广泛认知已建立；共识基础B：域名的跨域性易于低成本打通数据跨平台互通新与旧。
2.邮址的格式格式天然工整，对齐，具有域义。在此建议，是时候给星球们分配域名了。快邮件走即时通讯，慢数据跨天体的信息则符合古典人类的“寄信、收信”体验，大道至简。
3.物体世界的逻辑表达严谨清晰，万物互联的信息量比较人类数字社交社会所产生出的，可远大了几个数量级不止，并且日志更规整、明确。除了共识约束，人类应当以邮址涵义做泾渭区分，以认知为根基划出红线，

总结：自然界已经进化到了人工智能的时代，人们很快将有更多能力与精力向地球外看去（或者是不得已，只好必须向太空看去了）。现如今，人类对太空想法非常混沌，少数几个点亮点都在国家级和大公司，有切合实际的，也有不切合实际的，总成果是肤浅皮毛——因人类太空进程太慢。随着环境沉淀，一切将明确无误地指南人类：未来，就在太空，理应顺应这趋势。
